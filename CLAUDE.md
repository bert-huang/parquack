# Parquack — Claude context

## Build commands

```sh
npm run build:firefox       # one-shot Firefox build → build/firefox/
npm run build:chrome        # one-shot Chrome build  → build/chrome/
npm run watch               # watch + Firefox (sourcemaps, no minify)
npm run package:firefox     # build + zip → build/firefox/parquack.zip
npm run package             # both targets
npm run run:firefox         # launch in Firefox via web-ext
```

## Key conventions

- **Single source of truth for version**: `package.json` → `build.mjs` reads it at build time and injects into the manifest. Never hardcode the version elsewhere.
- **No manifest.json in source**: the manifest is generated by `buildManifest()` in `build.mjs`. Edit it there.
- **Build output**: `build/firefox/` and `build/chrome/`. These are not committed.
- **`apache-arrow` stays at `17.x`**: `@duckdb/duckdb-wasm` requires `apache-arrow@^17`. Do not upgrade to a newer major.

## Source layout

```
src/
  viewer.js          main UI logic (~1350 lines)
  viewer.html        page shell
  viewer.css         all styles
  background.js      toolbar click + DNR redirect rule
  icon.svg           source icon (rendered to PNG at build time)
  patches/

```

## AMO (Firefox submission) notes

- **`duckdb-browser-eh.worker.js`** contains a `Function` constructor from the DuckDB WASM binary. This is unavoidable — add a reviewer note: *"The `Function` constructor warning in `dist/duckdb-browser-eh.worker.js` originates in the pre-compiled @duckdb/duckdb-wasm binary, not the extension's own code."*
- The `apache-arrow/builder/valid.mjs` `new Function` is patched out via the `patchArrowValid` esbuild plugin (redirects to `src/patches/arrow-builder-valid.mjs`).
- `data_collection_permissions: { required: ["none"] }` declares no data collection. `"none"` is the official Mozilla sentinel value.
- `strict_min_version: "140.0"` (desktop) and `gecko_android.strict_min_version: "142.0"` — required because `data_collection_permissions` was introduced in those versions.

## DuckDB usage pattern

Files are registered as `current.parquet` in an in-memory DuckDB instance regardless of actual format. The loader detects extension/content-type and calls `read_parquet`, `read_json_auto`, or `read_csv_auto` accordingly. All queries run against the alias `data` (a view over the registered file).

## UI state

All mutable UI state lives in the `state` object at the top of `viewer.js`. Key fields: `loaded`, `fileType`, `page`, `totalRows`, `queryPage`, `queryTotalRows`, `queryPageRows`, `rowGroupId`.
